# うに ✦ けいさん あそび 仕様書

## 1. 概要

幼児～低学年向けの、簡単な計算式（四則演算）を学ぶ Web ベースのミニゲーム。
キャラクター「うに」の世界観と、既存の「うに ✦ もじ あそび」のテイスト（カラー / フォント / 触手アニメーション / 「うに」効果音 / 紙地のテクスチャ）を完全に踏襲する。

- 単一HTMLファイル (`uni-math.html`) で完結
- スマホ縦持ち優先、PC でも動作
- フレームワーク不要・依存なし（Google Fonts のみ外部）

## 2. ゲーム構造

### 2.1 全体フロー

```
[スタート画面 (モード選択)]
   ↓ 「はじめる」タップ
[1問目] → [2問目] → … → [10問目] → [結果画面 N/10]
                                            ↓ タップで再開
                                          [1問目] (同モード) → …
```

- **スタート画面でモード選択可能**（かんたん／ふつう／むずかしい）
- **各モードで全 10 問を出題**
- 10 問終了時点で **結果画面**（正答率＋メッセージ）を表示
- タップで同モードで再スタート
- ゲーム中のモード切替ボタンを押すと、いつでも進捗をリセットして 1 問目から再開

### 2.2 スタート画面のモード選択

スタート画面に 3 つのモードボタンを並べ、デフォルトで 1 つが選択された状態で表示する：

| 過去の達成状況 | デフォルト選択 |
| --- | --- |
| なし（初回など） | **かんたん** |
| かんたん で 10/10 達成済み | **ふつう** |
| ふつう で 10/10 達成済み | **むずかしい** |
| むずかしい で 10/10 達成済み | **むずかしい**（最上位なので維持） |

- 達成済みモードのボタンには **クラウンマーク（✦）** を右肩に表示
- ユーザーは選択を上書き可能（タップでデフォルトと違うモードを選べる）
- 選択中モードはオレンジ反転で表示

### 2.3 ベストスコアの永続化（localStorage）

```js
key:    "uni-math-best-scores-v1"
value:  JSON { easy: number, normal: number, hard: number }
```

- 10 問終了時、`saveBestScore(currentMode, score)` で過去最高得点を更新
- スタート画面の表示と「デフォルトモード」の判定に使用
- localStorage が使えない環境ではフォールバックで全 0（毎回かんたん始まり）

### 2.4 進捗表示

画面上部のモードボタン直下に **進捗ドット 10 個**を表示：

| 状態 | 見た目 |
| --- | --- |
| 未回答 | 薄いオレンジの小さな円 |
| 現在の問題 | 太い枠で拡大＋鼓動アニメ |
| 正解済み | オレンジ塗り＋発光 |
| 不正解／タイムアップ済み | 破線枠 |

### 2.5 共通ルール

各問題は「`A 演算子 B = C`」の式で表される。式中の **A / B / C / 演算子（X） の 4 つの位置のうち 1 つがランダムに「?」（穴）** となり、プレイヤーは画面下のパネルから正しい値をタップして穴を埋める。

- 制限時間：**10 秒**（カウントダウン数値は表示しない／タイマーバーで進行を抽象表示）
- 残り **3 秒** を切るとパネルがぐらぐら揺れ、タイマーバーが点滅し、ティック音が鳴る
- タイムアップ時も含め、問題終了時には **正解の式を音声で読み上げる**（例：「3 たす 5 は 8」）

### 2.6 モード仕様

| 項目 | かんたん | ふつう | むずかしい |
| --- | --- | --- | --- |
| 演算子の種類 | `+` のみ | `+` `-` | `+` `-` `×` `÷` |
| 演算子（X）が穴になるか | ならない（固定表示） | なる | なる |
| 穴の候補 | A / B / C | A / B / C / 演算子 | A / B / C / 演算子 |
| **A / B / C に 0 が出るか** | **出ない（A,B,C すべて 1 以上）** | 出る | 出る |
| C の値 | 必ず 1 桁 (1–9) | 必ず 1 桁 (0–9) | 1 桁または 2 桁 (0–99) |
| C が 2 桁の時の C の穴扱い | — | — | C は穴にしない |
| **演算子穴で A または B に 0 が含まれるか** | — | **含まれない** | **含まれない** |
| **演算子穴で曖昧解（複数演算子で同じ C）を許すか** | — | **許さない** | **許さない** |
| 数字穴の選択肢数 | 基本 2 択（たまに 3 択） | 3 択 | 4 択 |
| 演算子穴の選択肢数 | — | 2 択 | 4 択 |
| 数字選択肢の値域 | **1〜9** | 0〜9 | 0〜99 |

### 2.7 演算子穴の曖昧性除外（重要）

**演算子穴のとき、次の場合は op を穴の候補から外す**（A／B／C の数字穴に振り分ける）：

1. A または B に 0 を含むケース  
   例：`5 ? 0 = 5` は `+` でも `-` でも成立 → 不適切

2. 複数の演算子で同じ C になるケース  
   例：`6 ? 1 = 6` は `×` でも `÷` でも成立 → 不適切

実装：すべての候補演算子で `A op B = C` を計算し、ヒット数 ≥ 2 なら op 穴を作らない（`opHoleAmbiguous()`）。
これによりむずかしいモードの op 穴問題でも常に**正解が一意**になる（10000 問検証済み・曖昧件数 0）。

### 2.8 数字生成ルール

| 演算子 | 生成方法（かんたんモード時） | 生成方法（ふつう／むずかしい） |
| --- | --- | --- |
| `+` | A = 1–8、B = 1–(9−A)、C = A+B | A 0–9、B 0–9、C ≤ 9 制約あり時は B ≤ 9−A |
| `-` | A = 2–9、B = 1–(A−1)、C = A−B | C 1桁制約あり時：A 0–9、B 0–A／なし時：A 1–18、B 0–A |
| `×` | (かんたんでは出ない) | A, B 共に 0–9（C は最大 81） |
| `÷` | (かんたんでは出ない) | B 1–9、C 0–9 をランダム選び A = B × C。**割り算は必ず整数** |

### 2.9 選択肢の生成

- **正解値**を必ず含める
- **数字穴**: 正解値 ± 1〜5 のランダム値で、重複と範囲外を除いて埋める
  - かんたん: **1–9** 範囲（0 は出さない）
  - ふつう: 0–9 範囲
  - むずかしい: 0–99 範囲
- **演算子穴**: そのモードで使われる演算子からランダムに不正解を選ぶ
- 最後にシャッフル

## 4. 演出（モーション・音）

### 4.1 にぎやかしの背景うに（3体）

- 共通クラス `Uni`（既存の「もじ あそび」を踏襲）
- 背景うにの初期設定：**`idleMotion = false`** ──── アイドル時はアニメーションせず静止
- **タップ時**: 4 アクションから **ランダムに 2 つを組み合わせて発動**
  1. **jump**: 上下に飛び跳ねる（sin 波）
  2. **tentacle**: 触手 1 本がランダムに伸びる
  3. **rotate**: 1 周回転（左右ランダム）
  4. **scale**: 拡大 → 縮小
- **正解時**: 3 体が時差付き（80ms / 190ms / 300ms 後）でそれぞれ `triggerActions()` を発動して喜ぶ

### 4.2 正解時の降下うに

降下数：**`C` の値 ＋ 選択肢の合計値**（記号の選択肢は 1 個あたり 9 として計算）

例：
| 問題 | 選択肢 | 降下数 |
| --- | --- | --- |
| `?+2=3` | `[5, 1]` | 3 + 5 + 1 = **9 体** |
| `5+3=?` | `[8, 6, 3]` | 8 + 8 + 6 + 3 = **25 体** |
| `?−3=4` の演算子穴 | `[+, -]` | 7 + 9 + 9 = **25 体** |

クランプ：**最低 3 体・最大 60 体**（画面混雑とパフォーマンス考慮）

スポーン間隔 55ms。各うにはランダムなサイズ・回転速度・横揺れ・落下速度。
**各うには 1〜9 の数字を 1 つずつ持っており、タップでキャッチできる**（詳細は §5 結果画面 のキャッチ機能を参照。通常の正解時にも適用される）。

### 4.3 降下うにの物理（タイマーバー衝突）

降ってきたうにが**タイマーバーの「残り時間 fill 部分」と重なると物理的に跳ねる**：

- 反発係数 0.55 で `vy` 反転
- 横方向に ±90 のランダム速度を加える（散らばり）
- 回転速度 1.25 倍に加速
- **3 回跳ねる、または反発後の `|vy| < 70`** で疲弊し `passThrough` モードに入り、`life` を 1.6 秒に短縮して自然消失
- バーの **fill 部分のみ**が当たり判定（残り時間が短くなると当たり判定も短くなる）

### 4.4 残り 3 秒の警告

- タイマーバーが赤系で点滅（CSS animation: `timer-pulse`）
- 全選択肢ボタンが `shake` クラス付与でぐらぐら（位相ずらしで賑やか）
- `playUrgentTick()` で電子音を 1 秒に 1 回再生

### 4.5 答え合わせ演出（中央表示）

| 状況 | 中央表示 | サイズ | 進行方法 | 効果音 / 擬音 |
| --- | --- | --- | --- | --- |
| **正解** | 「せいかい」 | 大（88px / モバイル 64px） | **画面タップ または 3 秒で自動** で次の問題へ | `playCorrect`（ピンポーン×2）+ `speakOnomatopoeia('ピンポンピンポン')` |
| **不正解（タップ）** | 「ざんねん」 | 小（40px / モバイル 32px） | **答え読み上げ完了 + 最低 1.7 秒 を待って自動進行** | `playWrong`（ブブー、低音 2 連）+ `speakOnomatopoeia('ブブー')` |
| **タイムアップ** | 「タイムアップ」 | 小（40px / モバイル 32px） | **答え読み上げ完了 + 最低 1.7 秒 を待って自動進行** | `playTimeUp`（デロローン、下降グリッサンド + ビブラート）+ `speakOnomatopoeia('デロロン')` |

#### 共通の追加演出

- 正解パネルがオレンジに反転＋scale up・他はフェード
- 穴に正解値が `filled-correct` / `filled-wrong` / `filled-timeout` クラスで埋められる
- `speakAnswer()` で式を読み上げ（擬音の後にキュー再生）
- 進捗ドットが正解／不正解で塗り分けられる

#### 正解時のみの追加演出

- 「せいかい」ポップアニメーション（cubic-bezier の弾むイージング、回転付き）
- 「タップで つぎへ」のヒントが下部にバウンス表示
- 背景うに 3 体が時差で `triggerActions()`
- 降下うにスポーン

### 4.6 サウンド

| 関数 | 用途 | 音色 |
| --- | --- | --- |
| `playUni()` | うにのタップ時・降下時 | sawtooth + square + bandpass で「う」「い」を発声 |
| `playCorrect()` | 正解 | sine 1320Hz→1050Hz の「ピンポーン」を 2 回 |
| `playWrong()` | 誤答 | sawtooth + LPF で 220Hz / 195Hz の「ブブー」 |
| `playTimeUp()` | タイムアップ | triangle 520Hz→140Hz 下降グリッサンド + LFO ビブラート |
| `playUrgentTick()` | 残り 3 秒の警告 | square 880Hz パルス |
| `speakOnomatopoeia(text)` | 擬音発声（ピンポンピンポン / ブブー / デロロン） | `SpeechSynthesis API` (ja-JP)、rate 1.15 / pitch 1.4 |
| `speakAnswer(problem)` | 答え読み上げ | `SpeechSynthesis API` (ja-JP)、`〇 たす/ひく/かける/わる △ は □` 形式。**擬音の後にキュー再生** |

## 5. 結果画面（10 問終了時）

10 問終了すると結果オーバーレイが表示される：

```
        7
       ─────  /10
        やったね！

      タップで もういっかい
```

### スコア別メッセージ

| スコア | メッセージ |
| --- | --- |
| **10/10** | すごいね！ ぱーふぇくと！ |
| **8〜9** | とっても じょうず！ |
| **6〜7** | やったね！ |
| **4〜5** | もうすこし！ |
| **1〜3** | つぎは がんばろう！ |
| **0** | がんばろう！ |

### 演出

- **メッセージを音声で読み上げ**（`speakOnomatopoeia(msg, { rate: 1.05, pitch: 1.25 })`）
  - 「ピンポン」効果音は鳴らさない（仕様変更）
- **うにを無限に降らせる**（90〜170ms 間隔で `spawnSingleFallingUni`）— タップで再開するまで止まらない
- 背景うに 3 体は時差付き（0 / 140 / 280ms）でアクション

### 降下うにのキャッチ機能（結果画面 + 通常の正解時にも適用）

降ってくるうには **タップで捕まえて指で動かせる**。

| 操作 | 動作 |
| --- | --- |
| **降下うにをタップ** | 物理停止して指に追従。うにの上に **持っている数字 (1〜9 のランダム)** が大きく表示される |
| **指を動かす** | うにと数字が一緒についてくる |
| **指を離す** | うには **ランダム方向に飛び散る**（初速 260〜480 px/s）。バーをすり抜けて消失。**数字はその場に残り 3 秒かけてフェードアウト**（軽い重力で少し舞い上がってから落下） |
| **マルチタッチ** | 各指（pointerId）で個別のうにを同時に持てる |

実装メモ：
- 結果オーバーレイ全体は `pointer-events: none`、結果テキストとヒント部分のみ `pointer-events: auto`
- これにより、結果画面の文字以外の領域をタップした場合は下の canvas にイベントが届き、降下うにをキャッチできる
- 文字をタップすると再スタート

### 進行

- **タップで再スタート**（同モードで進捗リセット → 1 問目）
- 自動進行はなし（じっくり結果を見せる）

## 6. 状態遷移

```
[idle] ──startNextProblem()──▶ [active] ──tap or timeout──▶ [resolving]
   ▲                                                            │
   │       正解 → 「せいかい」表示 → タップ or 3 秒          │
   │       不正解 / タイムアップ → 小表示 → 読み上げ完了+1.7s │
   │                          ↓                                │
   │              progress.num >= 10 ?                          │
   │             ┌──────┴──────┐                              │
   │           No │              │ Yes                          │
   │             ↓              ↓                              │
   └──── 次の問題      [final] (タップで再スタート) ────────┘
```

| フェーズ | 説明 |
| --- | --- |
| `idle` | 問題切替の隙間 |
| `active` | 問題提示中（タイマー稼働、タップ受付） |
| `resolving` | 答え合わせ中（パネルロック・読み上げ・うに降下・中央表示） |

モード切替ボタン押下時は、進行中の問題と結果オーバーレイをキャンセル、進捗をリセットして 1 問目から再開。

## 9. アクセシビリティ・操作性

- **タッチ操作前提**: `touch-action: none`、`overscroll-behavior: none` で iframe / iOS Safari でのページスクロールを抑止
- **ポインタイベント** (`pointerdown` 等) で統一しタッチ・マウス両対応
- **タップ領域**: 選択肢ボタンは 84×84（モバイル 70×70）で十分大きい
- **音声**: `SpeechSynthesis` 未対応環境でも進行はブロックされない（フォールバック）
- **AudioContext**: ユーザー操作後に初期化することで iOS Safari の自動再生制限を回避
- **タップで進行**: 正解時の中央表示はオーバーレイ全体がタップ領域なので押し損じづらい
- **3 秒自動遷移**: 子供が「タップで進める」と分からない場合の救済として、正解時は 3 秒で自動進行
- **ダークモード対策（重要）**: スマホのダークモードで文字が見えなくなる問題を防ぐため、以下を多重に指定：
  - `<meta name="color-scheme" content="light only">` / `<meta name="supported-color-schemes" content="light">`
  - `<meta name="theme-color" content="#f7f1e8">`（モバイルブラウザのUIバー色も統一）
  - CSS で `html { color-scheme: light only; }` および `:root { color-scheme: light only; }`
  - `* { forced-color-adjust: none; }` で Windows ハイコントラストモード対策
  - `button, input, select, textarea` に明示的な `color: inherit; background-color: transparent;` を当てて、ブラウザが OS 配色を勝手に当てるのを抑止
  - `html, body` の背景・文字色は CSS 変数の **フォールバックとしてリテラル値** も併記（`background-color: #f7f1e8; background: var(--cream);`）
  - **これによりライト／ダークいずれの OS 設定でも、設計通りの「クリーム色 + 濃茶」配色で表示される**

## 11. 既知の判断事項（仕様明記なし → 実装で補完したもの）

| 項目 | 採用した実装 |
| --- | --- |
| **op 穴の曖昧性除外（仕様の補強）** | ユーザー指示は「A/B に 0 を含めない」だが、本質は「正解が一意になる」こと。実装では `B=1` で `×` と `÷` が同値になるケースなども検出して op 穴から除外（`opHoleAmbiguous()`）。10000 問検証で曖昧件数 0 |
| **localStorage キー名** | `uni-math-best-scores-v1`。将来形式変更時に `-v2` に上げて古いデータは無視できる構成 |
| **localStorage が使えない環境** | try/catch でフォールバック。読み込み失敗時は全 0 を返す（=毎回かんたんスタート） |
| **クラウンマーク** | スタート画面のモードボタンに 10/10 達成済みの印として `✦` を右肩に表示 |

| キャッチ判定の半径 | うにの半径 × 1.6（タップしやすさ重視） |
