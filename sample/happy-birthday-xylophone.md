# happy-birthday-xylophone

写真の鉄板木琴 (子供用おもちゃ) を Canvas 2D で再現し、 ユーザ提供の F5 録音サンプルをピッチシフトして
複数曲を自動演奏する楽器デモ。 v1=ピアノ から始まり v5=木琴 + 曲選択 + 半音バーまで進化したログ。

最終ファイル: HTML (ソース未配置、 ユーザがあとで sample/ に置く想定)

## 最終形 (v5) — 写真準拠の木琴

### レイアウト

- 縦長の **台形ベイマツ材フレーム** (上=低音、 下=高音、 下に行くほどバーが短く、 フレーム右端も斜めに細くなる)
- 左列に **自然音バー 12 本** (F5〜C7、 並びは F G A B C D E F G A B C)
- 右列に **半音バー 8 本** (F♯ G♯ A♯ C♯ D♯ F♯ G♯ A♯、 B-C 間と E-F 間はスキップ)
- 3 本のサポートレール (濃いオーク色) — 左縦・中央縦・右が斜めに走る
- 各バーに銀色の十字ねじ × 2 (ラジアルグラデで立体感)
- ラベルはオフセットシャドウを重ねたエンボス風刻印

### 配色 (写真に合わせる)

| 音 | 低音オクターブ | 高音オクターブ |
|---|---|---|
| F | 赤橙 | オレンジ |
| G | 深紅 | 赤 |
| A | ワインパープル | パープル |
| B | シルバー | シルバー |
| C | 青 | 濃紺 |
| D | 緑 | — |
| E | 金黄 | — |

## 音源 (サンプルベース)

ユーザ提供の 12.69 秒録音 `新規録音 5.m4a` から抽出した F5 (701 Hz, +7 cent) を基準サンプルとして使う。

### 録音処理

- 24 箇所の打音を検出 → 1.395 s 地点の F5 を採用
- FFT ピーク: 700 / 1927 / 3793 / 5453 / 6333 Hz (1×, 2.756×, 5.404×, 8.93× の典型的非整数倍音)
- ノイズ除去:
  - 80 Hz ハム → 200 Hz ハイパスフィルタ
  - 環境ノイズ → 12.45 s 付近の無音区間からプロファイル取得 → スペクトル減算ゲート
- 「曇り」 対策:
  - Low-shelf -3 dB @ 350 Hz (中低域の濁りカット)
  - High-shelf +4.5 dB @ 3 kHz (高域を持ち上げ煌めき追加)
- エンコード: 22 kHz / 80 kbps MP3 → base64 (約 9 KB) で HTML に埋め込み

### 再生方式

Web Audio API で MP3 を decode し `AudioBufferSourceNode.playbackRate = targetFreq / 698.46` で各音にピッチシフト。
半音は係数 ×1.0595。 サンプルベースなので倍音構造・減衰・打撃ニュアンスがそのまま再現される。

## アニメーション / 演出

- 押鍵時: 発光ハロー + 同心円リップル + 数 px 下に沈む
- 減衰: サインカーブ × 指数で上下振動 (`A · exp(-t) · cos(ωt)`)
- 中央下に再生中の音名表示 (♯ 対応)

## 曲選択 UI

- タイトル右 (モバイル時は下) に 4 つのチップボタン
- タップで切り替え再生、 現在再生中の曲がハイライト
- 切り替え時は **中断トークン** で前の再生を停止

## 操作

- ▷ play で自動演奏
- 各バーをタップ/クリックで手動演奏 (半音バーも鳴る)

## 進化ログ (要点)

| 版 | 内容 | 結果 |
|---|---|---|
| v0 | WebGL (Three.js) ピアノ | Three.js ロード失敗で描画されず |
| v1 | Canvas 2D ピアノ | 外部 CDN ゼロで確実描画。 2 オクターブ (C4〜B5)、 6 倍音加算合成音 |
| v2 | 木琴 (sin 振動 + 火の粉) | 短く明るい音、 マリンバ式に黒鍵が奥の段 |
| v3 | 鉄琴 (C=F ラベルシフト) | 全アルファベット 3 つシフト指定 → ユーザ確認で **誤解釈**、 C=ド に戻す |
| v4 | 写真に合わせた木琴 v2 + F5 サンプル音 | サンプルベース再生に転換、 v2 はバー 1 列構成 |
| v5 | 写真準拠 v3 — 縦型 + 半音バー追加 + 4 曲選択 + 夕べのさえずり placeholder | 最終形 |

## 設計上の学び (転用可能)

- **WebGL 依存は脆い** — 外部 CDN なしの Canvas 2D が確実 (v0 → v1 で外す決定)
- **サンプル + ピッチシフト > 加算合成** — 実楽器の響きを忠実に出すなら録音 1 音を基準にしたサンプラ方式
- **C=F のような全体ラベルシフトは誤解しやすい** — 周波数と表記の対応はユーザに必ず再確認
- **未確認の曲は placeholder + ユーザ確認** — 楽譜にアクセスできない時は推測で埋めず、 明示的に確認を求める
