# uni-math

[`uni-writing-game`](./uni-writing-game.md) と同じ「クリーム色の紙地 + 濃茶インク + うにオレンジ」 テイストで作る
**計算ドリル** コンテンツ。 `A 演算子 B = C` の式から 1 つを隠してタップで正解パネルを選ぶ。

## 最終形 (v8) の仕様

### コンセプト

- 「うに」 (Claude スターバーストロゴ、 [[project-uniland]] のマスコット) が見守る計算ドリル
- 1 問あたり 10 秒、 10 問で 1 セッション → スコア発表
- 3 モード (かんたん / ふつう / むずかしい) でルールと選択肢数を変える
- 視覚テイストは [`uni-writing-game`](./uni-writing-game.md) 完全踏襲

### モード差分

| 項目 | かんたん | ふつう | むずかしい |
|---|---|---|---|
| 演算子 | `+` のみ | `+` `−` | `+` `−` `×` `÷` |
| 演算子穴 | 出ない (固定表示) | 出る | 出る |
| 数字穴 | A / B / C のランダム 2 | A / B / C のランダム 2 | A / B / C のランダム 2 |
| C 桁数 | 1 桁固定 | 1 桁固定 | 最大 2 桁 (2 桁時は C が穴にならない) |
| 0 の扱い | 完全除外 | 通常通り | 通常通り |
| 選択肢数 (数字穴) | 2 (まれに 3) | 3 | 4 |
| 選択肢数 (演算子穴) | — | 2 | 4 |
| ÷ の整数性 | — | — | 必ず整数になる組のみ生成 |

### 問題生成の妥当性ガード

- **演算子穴 A=0 or B=0 を除外** — `0+B`/`0-B`/`0×B`/`0÷B` 等で複数演算子の結果が一致して曖昧になるため
- **op-hole 曖昧性チェックを汎用化** — `A op B === C` を満たす演算子が候補中 2 つ以上ある問題は生成しない
  - 例: `6 × 1 = 6` と `6 ÷ 1 = 6` の同時成立 (B=1) を排除
  - 10,000 問の Node 検証で normal / hard ともに **曖昧件数 0**
- **かんたんは 0 完全排除** — `A = 1..8, B = 1..(9-A), C = A+B` で最小 1。 5,000 問検証で 0 出現 0 件

### タイマー

- 1 問 10 秒制限。 カウントダウン数値は表示しない
- **残り 3 秒** からパネル全体に shake アニメーション、 タイマーバー点滅 + 電子ティック音
- 時間切れ → タイムアップ扱い

### 正解 / 不正解 / タイムアップ表示

| 結果 | 中央表示 | 進行 |
|---|---|---|
| 正解 | 「せいかい」 88px ポップアニメ | タップで即進行 or 3 秒で自動進行 |
| 不正解 | 「ざんねん」 40px 小表示 | 答え読み上げ完了 + 最低 1.7 秒で進行 |
| タイムアップ | 「タイムアップ」 40px 小表示 | 答え読み上げ完了 + 最低 1.7 秒で進行 |

オーバーレイ背景は `pointer-events: none` で透明、 テキスト部分だけ `auto`。 これによって下の canvas (降下うに)
へのタップが通る + 文字タップで再スタート両立。

### 効果音 + 音声

| 結果 | 効果音 | 擬音 (SpeechSynthesis) |
|---|---|---|
| 正解 | 「ピンポーン」 ×2 ベル音 (sine 1320 Hz → 1050 Hz) | 「ピンポンピンポン」 |
| 不正解 | 「ブブー」 (sawtooth 220 Hz/195 Hz, LPF, 2 連) | 「ブブー」 |
| タイムアップ | 「デロローン」 (triangle 520 Hz → 140 Hz 下降 + LFO ビブラート) | 「デロロン」 |
| 残り 3 秒 | 電子ティック音 | — |

答え読み上げ (例: 「3 たす 5 は 8」) は擬音と同じキューに乗せて連続再生。 `SpeechSynthesisUtterance.onend` で
完了検知 + 最低 1.7 秒表示の **両方** を満たしたら進行 (壊れた環境向けに 5 秒タイムアウトフォールバック)。

### うに (キャラクター)

#### 背景うに ×3 (賑やかし)

- 問題式の周囲に 3 体配置
- `idleMotion = false` — 普段は静止
- タップ → `{ jump, tentacle, rotate, scale }` から重複なし 2 つランダム選択して同時実行
- **正解時** に 80 / 190 / 300 ms の時差で `triggerActions()` 発動して喜ぶ

#### 降下うに (Falling Uni)

正解時に画面上から落ちてくる。 各個体は 1〜9 のランダムな数字を持つ。

- **降下数の計算式 (重要):**
  - 通常: `C の値 + 選択肢の合計値` (演算子は 1 個あたり 9 とみなす)
  - 例: `x + 2 = 3` で選択肢 `[5, 1]` → `3 + 5 + 1 = 9` 体
  - 下限 3 体保証、 上限 60 体クランプ (むずかしいモードで爆発しないように)
  - スポーン間隔 55 ms
- **タイマーバー衝突物理:**
  - 反発係数 0.55、 横方向散逸 + 回転加速
  - 3 回バウンス or 速度低下で `passThrough` モード → 1.6 秒で自然消失
  - 残り時間が短いとバー幅が縮み、 当たり判定もそれに連動
- **キャッチ機能:**
  - canvas に `pointerdown/move/up` + `setPointerCapture` で指追従
  - キャッチ中はうに物理停止 + 上に大きな数字 (白縁取り + オレンジ塗り)
  - **マルチタッチ対応** (pointerId 単位で個別管理)
- **指を離した時:**
  - うには 260〜480 px/s のランダム方向に飛び散る (やや上向きバイアス)
  - バー passThrough + 回転加速
  - `FloatingNumber` クラスが数字をその場に残す: 軽い重力で少し舞い上がってから落下、 3 秒でフェードアウト
  - 太字 + 白縁取りで降下うにに紛れても視認可能

#### 結果発表中の **無限降下**

- `startEndlessRain` / `stopEndlessRain` で 90〜170 ms 間隔で無限スポーン
- タップ再スタート時 / モード切替時に確実に停止

## スタート画面 + モード解放

- スタート画面に 3 つのモードボタン (かんたん / ふつう / むずかしい)
- **デフォルト選択ロジック:**
  - 何も達成なし → かんたん
  - かんたん 10/10 達成 → ふつう
  - ふつう 10/10 達成 → むずかしい
  - むずかしい 10/10 達成 → むずかしい (最上位維持)
- 達成済みモードはボタン右肩に **✦ クラウンマーク**
- ユーザは選択を上書き可能

### localStorage 永続化

```
key:   "uni-math-best-scores-v1"
value: JSON { easy: number, normal: number, hard: number }
```

`saveBestScore(mode, score)` で過去最高得点を更新、 `loadBestScores()` で起動時読み込み。
プライベートブラウズ等で `localStorage` が使えない時は try/catch でフォールバック (全 0)。

## ダークモード対策 (重要パターン)

最終的に有効だった方法:

### 失敗した「保険を重ねる」 方式 (v7)

`meta name="color-scheme" content="light only"` + CSS `color-scheme: light only` + `forced-color-adjust: none` +
ボタン/フォーム色固定 + body リテラル + var() 併記 を全部入れても **完全には防げない** ことがある (特定モバイル環境)。

### 実際に効いた方式 (v8)

スタート画面 `.start-overlay` がダークモードでも崩れなかった理由が
**「要素自身が `position: fixed; inset: 0; background: <不透明色>` で画面全体を覆う独立した不透明レイヤーだった」**
ことに気付き、 ゲーム画面の `#stage` も同じパターンに揃える:

```css
#stage {
  position: fixed; inset: 0;
  color: #2a2118;
  background-color: #f7f1e8;       /* 不透明クリームを固定値で */
  background-image:
    radial-gradient(...),
    url("data:image/svg+xml;...");  /* 紙テクスチャ */
  isolation: isolate;               /* スタッキングコンテキスト分離 */
}
```

body は最小限のフォールバック (`background-color` のみ) に削減。
**「自前で画面全体を覆う不透明レイヤーを持つ要素」 を 1 つ用意するのが鍵** で、 body 経由の背景は OS のダークモード挙動の影響を受けて沈むことがある。

## 設計上の学び (転用可能)

- **「クリーム紙 + 濃茶 + うにオレンジ」 が UniLand の標準テイスト** — 別コンテンツでも踏襲が前提 ([[project-foundation_ui]] に統合候補)
- **計算問題の op-hole 曖昧性は局所チェックでは漏れる** — 候補演算子全部を試して 1 つしか満たさないことを確認するのが安全。 0 や 1 が絡む組合せに特に注意
- **モバイル/iframe のダークモード対策は「自前不透明レイヤー」 が最強** — color-scheme: light only や色固定の保険は完全ではなく、 `position: fixed; inset: 0; background: <opaque>; isolation: isolate;` の要素を 1 つ置くのが確実
- **進行の決定は「音声 onend + 最低表示時間」 の AND** — どちらかだけだと音声切れや早すぎ進行が発生する。 壊れた環境向けに最大タイムアウトも併用
- **オーバーレイの透過タップ** — `pointer-events: none` を背景に、 テキスト部分だけ `auto` にすると、 結果表示中も裏のゲーム要素 (キャッチ等) を触れる
- **降下数の計算式は「画面上の総和」 が直感的** — `C + 選択肢合計` (記号は 9 扱い) は学習者にも理屈が見える設計
- **`idleMotion` フラグはここでも使う** — uni-writing-game と同じパターン。 賑やかしキャラの普段静止 + イベント時動作は UniLand 標準
