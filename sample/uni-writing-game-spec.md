# うに もじ あそび — 仕様書

> Claude スターバーストロゴをモチーフにした「うに」キャラと一緒に、ひらがな・カタカナ・漢字の文字をなぞって覚える、子供向けのインタラクティブ Web ゲーム。

---

## 1. コンセプト

幼児〜小学校低学年が、**点線で示された文字をゆびでなぞる**ことで、ひらがな・カタカナ・小学3年生程度までの漢字に触れる。「うに」（Claude のスターバースト形ロゴ）が学習のお供として登場し、なぞりに合わせて触手を伸ばし、完成すると単語を読み上げる。

- **対象**: 文字に興味を持ち始めた未就学児 〜 小学校低学年
- **プレイ単位**: 1単語 (1〜6文字)
- **1セッションの目安**: 15秒〜1分／単語

## 3. ゲームフロー

### 状態機械

| 状態 | 説明 | 抜け条件 |
|------|------|---------|
| `idle` | 起動直後、スタート画面 | 「はじめる」ボタン押下 |
| `tracing` | 文字をなぞっている最中 | 全画なぞり完了 OR `AUTO_ADVANCE_SEC` 経過 |
| `completing` | 単語完成の演出 (0.6秒) | 演出終了 |
| `speaking` | 単語の読み上げ中 | `SpeechSynthesis.onend` または最大2秒 |
| `done` | 次の単語待ち | キャンバス上のタップ |

### 単語完成までの流れ

```
startNewWord()
   ↓ pickRandomWord (現在モードでフィルタ、直前と同じは避ける)
   ↓ prepareCurrentChar (画とパス点列を抽出)
   ↓ state = 'tracing'
   ↓
[ なぞり ]
   - 現在の画の始点にメインうに配置
   - 指の動きに合わせて触手が伸びる
   - 各点に近づくと通過済みに変わる
   - 全点通過 OR 10秒進捗なし → 次の文字
   ↓ 全文字完成
   ↓ state = 'completing'
   ↓ スパークル発火、単語パルス開始
   ↓ 0.6秒後 state = 'speaking', SpeechSynthesis 起動
   ↓ 読み上げ終了で state = 'done'
   ↓ ヒントが「タップで つぎへ」へ変化
   ↓ タップ → startNewWord()
```

## 4. メカニクス

### 4.1 なぞり (Tracing)

文字は連結成分ごとに分解し、各画(stroke)から **5〜12 個**のサンプル点を抽出してパスを構成する。

**可視化ルール**

| 要素 | 表示 |
|------|------|
| 通過済みの点 | 黒丸 (`#2a2118`, r=5) |
| 通過済みの線 | 黒い実線 (太さ14) |
| 現在の目標点 | パルスする橙丸 (外ハロー r=18, メイン r=10、内ハイライト r=3.5) |
| 現在の画の未通過点 | 半透明橙丸 (`rgba(217,119,87,0.55)`, r=3) |
| 未来の画の点 | より淡い橙丸 (`rgba(217,119,87,0.4)`) |
| 画の始点(未通過) | 通常より少し大きく (r=4.5) |
| 画と画の間の線 | 橙色破線 (`[2, 9]`、太さ4) |

**ヒット判定**
- 現在の目標点の中心からポインタまでの距離 < `HIT_RADIUS` (32px) で通過扱い
- 通過した点は永続的に「通過済み」になり、文字内では巻き戻らない
- 通過のたび: 小スパークル + インク雫 + 「うに」音 1回

**画の進行ルール**
- 現在の画の全点を通過 → 次の画へ移行 (`strokeIdx++`, `pointIdx=0`)
- 全画通過 → 文字完成

### 4.2 オートアドバンス

- 各文字に開始時刻からのタイマーがある (`charTimer`)
- 1点でも通過するごとにタイマーは0にリセット
- タイマーが `AUTO_ADVANCE_SEC` (10秒) を超えたら、強制的に次の文字へ

### 4.3 メインうに

- 現在の画の **始点** からやや左上 (`-36px, -18px`) に配置される
- 触手のうち、ポインタに最も近い角度のものが「リーチ触手」となり、ポインタ位置まで実際に伸びる
- リーチ触手は色が濃く (`#b85a3e`)、カーブを直線化して直接伸びる
- ポインタが非アクティブのときは通常のうに (全触手が wiggle)
- 画が変わると新しい始点に滑らかに移動 (時間ベースの指数スムージング、レート12)

### 4.4 背景うに (にぎやかし)

- キャンバスの3隅に小さな (`scale=0.55`) うにを3体配置
  - 左上 `(W*0.13, H*0.16)`
  - 右上 `(W*0.87, H*0.21)`
  - 下中央 `(W*0.50, H*0.86)`
- 通常時は普通に wiggle するだけ (タッチターゲットとしての視認性のみ)
- **タップ判定**: ポインタ押下時に各背景うにとの距離 < `50 * baseScale` (≈27.5px) でヒット
- ヒット時、{`jump`, `tentacle`, `rotate`, `scale`} から **重複なしで2つランダム選択**して同時実行
- アクション中は再トリガー不可 (`isPlayingActions()` で防止)

#### アクションの仕様

| アクション | 効果 | 数式 |
|-----------|------|------|
| `jump` | 上向きにポンと跳ねる | `yOffset = -50 * sin(πt)` |
| `tentacle` | ランダムな1本の触手が伸び縮み | 触手長 ×= `1 + sin(πt)` (最大2倍) |
| `rotate` | 体が1回転 (左右ランダム) | `rot = 2π * t * dir` (`dir ∈ {-1,1}`) |
| `scale` | 大きさが膨らむ | `scale = 1 + 0.4 * sin(πt)` |

すべて `t = 0 → 1` の0.65〜0.9秒で完了する。複数アクションは合成され、同時に効く。

## 5. モードフィルタ

タイトル直下に4つのフィルタボタン。文字種コードレンジで単語を分類し、選択モードに含まれる単語のみが抽選される。

| ボタン | 対象 | 判定 |
|--------|------|------|
| ぜんぶ | 全250語 | 制限なし |
| あ | ひらがなのみ75語 | 全文字が U+3040〜U+309F (またはー) |
| ア | カタカナのみ75語 | 全文字が U+30A0〜U+30FF または U+31F0〜U+31FF |
| 漢 | 漢字を含む100語 | 1文字以上が U+4E00〜U+9FFF |

モード変更時、`done` / `speaking` / `idle` 状態なら即座に新しい単語を抽選し直す。`tracing` 中は中断せず、次の単語から適用される。

## 8. オーディオデザイン

### 「うに」効果音 (合成)

`AudioContext` で都度合成するフォルマント方式の "う" + "に" の2音節。

- ベースピッチ: 182〜196Hz (毎回ランダムジッタ)
- "う": バンドパス F1=320Hz, F2=800Hz, ハーモニックF3=700Hz
- "に": バンドパス F1=280Hz, F2=2200Hz, ハーモニックF3=1500Hz
- ノコギリ波 + 矩形波の重ね、`tanh` シェイパーで丸める
- 音量エンベロープ 12ms アタック → 90ms サスティン → 40ms リリース

**発火タイミング**
- なぞりで点を通過するたびに 1セット (`playUni()`)
- 背景うにのタップ時に高めピッチで 1セット (`playUni(40)`)

### 単語読み上げ

`SpeechSynthesis` API の `ja-JP` ボイス:
- `rate: 0.85`, `pitch: 1.05`
- 音声選択: `voice.lang.startsWith('ja')` の最初のもの
- `onend` 検知 + 最大2.5秒程度のタイムアウトのフォールバック

## 9. 技術実装

### アーキテクチャ概観

```
ブラウザ (single HTML)
├── HTML
│   ├── タイトル / モードボタン / 単語行 / キャンバス / ヒント
│   └── スタートオーバーレイ
├── CSS
│   ├── テーマ変数 (--cream, --uni 等)
│   ├── アニメーション (word-breathe, bob)
│   └── レスポンシブ (@media max-width 480px)
└── JS (ES2018+)
    ├── WORDS (250 配列)
    ├── 文字種判定 + wordsByCategory フィルタ
    ├── AudioContext 合成
    ├── 文字パス抽出 (オフスクリーンキャンバス)
    │   ├── テキスト描画 → アルファチャネル
    │   ├── 連結成分検出 (4近傍 BFS)
    │   ├── 縦→横の順でソート (画順の近似)
    │   └── PCA で各成分の主軸方向に等間隔サンプリング
    ├── Uni クラス (メイン/背景共通)
    │   ├── 触手 (11本) wiggle + リーチモード
    │   └── アクション系 (jump/tentacle/rotate/scale)
    ├── パーティクル系 (sparkles, droplets)
    ├── ステートマシン (idle/tracing/completing/speaking/done)
    └── Pointer Events によるトレース判定
```

### 文字パス抽出の詳細

1. オフスクリーン 220×220 キャンバスに `Zen Maru Gothic 900` で文字を描画
2. ピクセルアルファ > 128 を「インク」とする2値マスクを作る
3. 4近傍 BFS で連結成分を抽出 (12px以上のもののみ採用)
4. 各成分の最小Y座標で縦に並べ、近い場合は最小X座標で横に並べ替え (画順の近似)
5. 各成分について:
   - 重心と分散行列を計算
   - 第1主成分方向 (PCA) を求める
   - 全点をその軸に射影してソート
   - 上→下/左→右の方向にする (画順の通常の流れ)
   - 等間隔に5〜12点サンプリング (`min(12, max(5, comp.length/35))`)
6. 結果として `strokes: [[{x,y},{x,y},...], ...]` の構造を得る

**制限**: PCA ベースの近似なので、伝統的な書き順とは一致しないことがある。あくまで「点線をなぞる体験」を提供するもので、書き順学習用としては不正確。

### Uni クラス

```js
class Uni {
  constructor(x, y, scale)
  setTarget(x, y)            // 目標位置
  reach(targetX, targetY)    // 触手のリーチ先 (canvas絶対座標)
  unreach()                  // リーチ解除
  triggerActions()           // 2つのランダムアクションを開始
  isPlayingActions()         // 実行中かどうか
  update(dt)                 // 状態更新
  draw(c)                    // 描画
  hitTest(px, py)            // 円判定
}
```

- 11本の触手それぞれに独立した wiggle 位相、長さ、太さ、カーブ
- リーチ時は最も近い角度の触手が選ばれて伸びる (本体の rotate / scale も考慮した座標逆変換)
- アクション計算は `actYOffset` `actScale` `actRot` `actTentBoost` の4変数として draw に渡される

### Pointer Events ハンドリング

- `pointerdown` で active=true、座標記録、背景うにヒット判定
- `pointermove` で座標更新 (active時のみ)
- `pointerup` / `pointercancel` / `pointerleave` で active=false
- `touch-action: none` で iOS のスクロール抑止
- スペースキー/Enterで `done` → 次の単語

### メインループ

`requestAnimationFrame` から `dt` を計算し、すべてに渡す:

```
tick(t):
  dt = min(0.05, (t - lastTime) / 1000)
  clear canvas
  draw bg unis (背景レイヤ)
  update + draw particles
  draw path (点線、目標点、通過済み線)
  if tracing: updateTracing(dt)
  update main uni
  if tracing: draw main uni
  if completing: タイマー進行 → speaking 遷移
```

## 11. 既知の制限と今後の拡張余地

### 制限事項

- **書き順は正確ではない**: PCA ベースの近似なので、本来の書き順とは異なる場合がある (特に複雑な漢字)
- **複雑な画の検出**: 「は」「ば」など接続部分が太い文字は、複数画を1つの連結成分として扱ってしまう場合あり
- **長音符 ・ 濁点・半濁点**: 「ピ」の半濁点や「バ」の濁点は分離点として扱われ、独立した小さな画になる
- **モード「mixed」は抽選対象外**: 漢字+ひらがな混合の単語は今のリストに無いが、追加した場合は `kanji` モードで含まれる
- **オフライン音声合成**: ブラウザ・OSによっては日本語音声がインストールされていないと無音

### 拡張可能性

- **書き順データ**: KanjiVG など SVG ベースの書き順データを取り込めば、伝統的書き順をなぞれる
- **ふりがな表示**: 漢字単語に小さくひらがなを併記
- **進捗トラッキング**: なぞった単語の履歴、よく間違える文字の頻度学習
- **難易度調整**: HIT_RADIUS をモードで変える (低年齢=広め)
- **テーマ切替**: ダーク背景、暖色/寒色の他のパレット
- **複数言語対応**: 文字データのレンダリングが汎用なので、英字・ハングルなどにも転用可
